4	ТРЕБОВАНИЯ К ПРОГРАММЕ ИЛИ ПРОГРАММНОМУ ИЗДЕЛИЮ
4.1	Требования к функциональным характеристикам
4.1.1	Состав выполняемых функций
Программа должна быть разделена на логические модули, обеспечивающие автоматизацию ключевых процессов спортклуба:
1) Модуль управления доступом и персоналом
- Авторизация сотрудников с использованием хеширования паролей;
- Ведение реестра сотрудников: функции добавления новых записей, редактирования данных действующих сотрудников и их удаления (деактивации).
2) Модуль работы с клиентами
- Регистрация новых клиентов и ведение единой базы (анкета клиента);
- Поиск клиентов по ФИО или номеру телефона.
3) Модуль продаж и абонементов
- Формирование справочника типов абонементов (наименование, срок действия, стоимость);
- Оформление договора: привязка выбранного абонемента к конкретному клиенту.
- Регистрация факта оплаты, выбор способа оплаты и автоматическая привязка транзакции к номеру договора.
4) Модуль контроля посещаемости
- Фиксация визитов с помощью ключ-карты (время входа и выхода клиента);
- Проверка статуса абонемента при входе (активен/просрочен).
5) Модуль расписания индивидуальных занятий
- Создание записи клиента на тренировку;
- Просмотр актуального расписания, возможность редактирования времени/зала или отмены занятия.
6) Модуль учета инвентаря
- Список залов и закрепленный за ними инвентарь;
- Проверка состояния инвентаря (В ремонте/исправен);
- Принадлежность инвентаря к определенному залу.
7) Модуль отчетности
- Автоматическое формирование списка активных клиентов на текущую дату;
- Генерация отчета по выручке за произвольный период (день, месяц, год);
- Выгрузка журнала посещений для контроля трафика спортклуба.
4.1.2	Организация входных и выходных данных
Входные данные:
- Сотрудник: ID сотрудника, Фамилия, Имя, Отчество, Логин, Хеш пароля, ID должности, Телефон;
- Должность: ID должности, Название должности, Уровень доступа;
- Клиент: ID клиента, Фамилия, Имя, Отчество, Дата рождения, Телефон, E-mail, Паспортные данные (серия + номер – 10 цифр);
- Абонемент: ID абонемента, Название, Срок, Стоимость;
- Договор: ID договора, ID Клиента, ID Абонемента, ID сотрудника (тот, кто оформлял договор), ID оплаты, Дата начала, Дата окончания;
- Оплата: ID оплаты, Сумма, Дата оплаты, Статус, Способ оплаты, назначение оплаты;
- Посещение: ID Клиента, Дата и время прихода, Дата и время ухода;
- Зал: ID зала, Название, Вместимость;
- Расписание индивидуальных занятий: ID занятия, ID клиента, Дата и время, Название занятия, ID Зала, ID сотрудника (может быть только тренер), ID оплаты;
- Инвентарь: ID инвентаря, ID зала (где находится), Название, Статус (Исправен/На ремонте).
Выходные данные:
- Список активных клиентов: Список людей, имеющих действующие договоры на текущую дату;
- Отчет по выручке: Стоимость проданных абонементов за выбранный период;
- Журнал посещений: История визитов за день для контроля посещаемости.
4.1.3	Временная характеристика
- время отклика системы должно быть не более 3 секунд;
- система должна обеспечивать доступ к данным не более, чем за 3 секунд.
4.2	Требования к надежности:
- резервное копирование базы данных должно выполняться автоматически каждые 24 часа;
- ведение системного журнала, в котором фиксируется ID сотрудника, время и тип операции при изменении данных в таблицах «Договор» и «Оплата»;
- система должна блокировать ввод отрицательных значений для финансовых полей и дат рождения, превышающих текущую дату;
- разграничение прав доступа на уровне интерфейса;
- хранение паролей в зашифрованном виде;
- защита от SQL-инъекций;
- использование лицензированного программного обеспечения.
4.3	Требования к интерфейсу
Программный интерфейс должен соответствовать следующим требованиям:
- Весь текстовый контент и элементы управления должны быть на русском языке. 
- Интерфейс должен динамически изменяться в зависимости от роли авторизованного пользователя:
1)	Администратор: полный доступ к управлению персоналом, доступом и справочниками.
2)	Менеджер: функционал регистрации клиентов, оформления договоров и работы с оплатами.
3)	Тренер: доступ к расписанию индивидуальных занятий и списку закрепленных залов.
- Интерфейс должен корректно отображаться и сохранять работоспособность при разрешении экрана от 1366x768 пикселей и выше.
- Понятная навигация, обеспечивающая доступ к основным функциям системы не более чем в 3 клика.
- Единый стиль графических элементов (кнопок, полей ввода, шрифтов) во всех модулях системы.
4.4	Требования к составу и параметрам технических средств
Для нормального функционирования данной информационной системы необходим компьютер, клавиатура, мышь и следующие технические средства:
1) Серверная часть: 
- Процессор: Intel Core i3 / AMD Ryzen 3 (архитектура x64) с частотой не менее 2.4 ГГц.
- Оперативная память: не менее 8 Гб (с учетом работы ОС и СУБД).
- Свободное место на диске: 50 Гб (SSD рекомендуется для быстродействия БД). 
2) Клиентская часть (рабочее место сотрудника): 
- Процессор: Intel Core i3 / AMD Ryzen 3 (архитектура x64) с частотой не менее 2.4 ГГц.
- Оперативная память: не менее 4 Гб. 
- Свободное место на диске: 1 Гб. 
- Монитор: стандартный с поддержкой HDMI и разрешением от 1920x1080. 
3) Сетевое взаимодействие:
- Наличие локальной сети (LAN) со скоростью передачи данных не менее 80 Мбит/с для обеспечения стабильной связи клиента с базой данных на сервере.
4.5	Требования к информационной и программной совместимости 
Для полноценного функционирования данной системы необходимо наличие операционной системы Microsoft Windows 10 или выше. База данных должна быть реализована на Microsoft SQL Server 2019/2022 или выше. Программа должна быть написана с использованием Python версии 3.10 или выше и PyQt6 для разработки графического интерфейса.
4.6	Требования к маркировке и упаковке
Клиентское приложение поставляется в виде exe-файла или установщика, а серверная база данных разворачивается отдельно (скрипты создания БД, инструкции).
4.7	Требования к программной документации
 Документация поставляется в виде: Технического задания, пояснительной записки, руководства пользователя и руководства администратора.
4.8	Требования к тестированию
Тестирование включает в себя:
- проверка реализации всех функций, описанных в п. 4.1 (управление персоналом, регистрация продаж, ведение расписания и формирование отчетов); 
- проверка соответствия экранных форм требованиям п. 4.3, включая корректность отображения на различных разрешениях (от 1920x1080) и наличие разграничения прав доступа (администратор, менеджер, тренер);
- замер времени отклика системы при выполнении поисковых запросов и передвижению по системе;
- проверка корректности обработки ошибочных входных данных.
